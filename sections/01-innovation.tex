\section{作品难点与创新}

本作品面向户用储能、离网备用供电与远程状态监测等应用场景，围绕 GD32G553 主控、GD32C113 BMS 和 GD32VW553 无线模块构建多 MCU 协同系统。系统的设计重点不是将所有功能压缩到单一控制器中，而是通过明确分工、统一接口和状态机管理，实现采样、保护、通信、转发和展示等功能的协同运行。主控侧采用 APP / Middleware / Protocol / HAL 四层解耦结构，通过统一调度器和发布/订阅总线组织任务；BMS 侧采用测量、保护、FET、均衡和 SOC 模块分层实现，并以产品态机保证异常情况下的快速降级；无线侧则负责数据转发、状态展示和参数交互。该系统的核心价值在于，在保证本地安全控制优先级的前提下，提高系统可观测性、可维护性和工程扩展能力。

系统采用 GD32G553、GD32C113 和 GD32VW553 三颗 MCU 分工协作。其中，GD32G553 作为主功率控制器，负责光伏输入功率变换、储能充放电控制、直流母线调节和交流输出控制；GD32C113 作为电池管理控制器，负责电芯采样、均衡、SOC 估算和故障保护；GD32VW553 作为联网控制器，负责无线通信、参数配置、运行日志和告警信息管理。该架构既保证底层功率环路的实时性，又提升了系统安全性和用户侧交互能力。

\subsection{应用场景与作品定位}

随着户用储能、分布式能源和便携式备用电源需求的增长，工程系统不仅需要完成基本的数据采集和控制动作，还需要同时满足安全保护、状态展示、远程通信和调试验证等多方面要求。工程作品的重点通常不仅在于单一功能是否实现，更在于系统架构是否清晰、功能链路是否闭合、模块之间是否解耦以及是否具备进一步扩展的能力。因此，本作品的定位并非某一个孤立器件或单一算法的演示，而是围绕“采样-判定-保护-通信-展示”这一完整链路，构建一套可运行、可监控、可调试、可扩展的多 MCU 协同系统。

从功能划分上看，GD32G553 主控工程承担系统调度、BMS 数据接收转发、PWM 输出和中间协议组织等任务。该工程采用分层结构，将底层硬件抽象、通信协议、数据总线和应用调度分开处理，避免了采样、协议、控制与通信逻辑混杂在同一层中造成的耦合问题。GD32C113 BMS 工程则负责电池测量、保护判断、FET 策略、均衡控制和 SOC 计算等安全相关功能，并通过 CAN FD 遥测向上位节点输出状态信息。无线模块的作用是将系统运行状态、告警信息和关键参数进一步传递到页面或上位端，便于观察和调试。

与传统单控制器集中式方案相比，本作品更强调控制链路的边界划分。主控负责系统级协同，BMS 负责电池侧安全，Wi-Fi 模块负责远程交互，三者之间通过统一的数据帧和状态码进行连接。这样设计的优点在于：一方面可以将对实时性要求最高的控制和保护逻辑保留在本地；另一方面可以将刷新频率较低、交互性较强的监控任务移出核心控制路径，从而保证整机运行的稳定性和可维护性。

本系统的目标不是单一变换器验证，而是完成从电源硬件、控制算法、电池保护到无线监控的整机方案。论文后续章节将围绕系统架构、主功率电路、控制策略、保护机制和测试验证展开。

\subsection{难点分析}

实现一个光伏、储能一体化的微型数字电源系统，主要面临以下难题：

\begin{enumerate}[label=\arabic*、, leftmargin=*, itemsep=0.8ex, parsep=0pt, topsep=0.5ex]
  \item \textbf{实时控制与非实时通信之间需要严格解耦。}  
  主控侧既要处理采样、状态判断、PWM 输出和保护动作等高实时任务，又要承担 BMS 数据转发、日志组织和无线通信等相对低实时任务。如果这些任务在同一层级中直接混合，极易造成中断响应受阻、主循环阻塞或控制周期抖动。因此，必须在架构上将实时控制与通信交互明确分离，保证核心控制任务不被监控任务干扰。

  \item \textbf{主控、BMS 与无线模块之间的职责边界必须明确。}  
  主控并不直接替代 BMS 完成电池安全判断，BMS 也不应越权承担系统级调度。主控侧负责系统状态组织、数据汇总和对外转发，BMS 侧负责电池采样、保护、均衡和 FET 策略，无线模块负责展示和交互。职责边界一旦不清晰，就容易出现重复判断、状态冲突或保护动作不一致的问题，因此必须通过统一协议和状态码加以约束。

  \item \textbf{数据协议、单位与状态定义必须统一。}  
  主控与 BMS 之间使用 CAN FD 遥测，主控与无线模块之间使用 UART 帧转发，若不同模块对电压、电流、温度、SOC、告警码和状态码采用不同单位或不同编码，就会造成页面显示、串口日志和控制逻辑之间的不一致。特别是在 BMS 遥测中，0x300、0x301、0x302、0x303、0x310 等报文需要明确字段布局、缩放单位和字节序；在主控转发中，UART AA55 帧也必须保持固定格式，才能保证系统级联调的稳定性。

  \item \textbf{测量与标定的准确性直接影响系统可用性。}  
  主控侧的 ADC 采样模块需要对电压、电流和温度信号进行统一采集，并完成零点校准、增益换算和物理量转换；BMS 侧又要完成电芯电压、PACK 电压、电流、温度和均衡状态的读取与判定。若标定方法不一致、换算关系不清晰或采样链路存在漂移，则会导致保护阈值误判、SOC 偏差或监控显示失真。因此，测量模块和标定机制必须与业务逻辑同步设计。

  \item \textbf{系统可观测性要求高，但又不能侵入核心控制链路。}  
  系统必须提供足够丰富的状态回显、运行日志和异常信息，同时又不能让这些信息采集与展示影响底层安全控制。因此，如何在不干扰主控和 BMS 核心链路的前提下完成数据组织、状态上报和远程显示，是本作品联调阶段的重要难点。

  \item \textbf{三端口功率变换对采样、驱动和保护协同要求高。}  
  三端口功率变换需要同时考虑光伏端、电池端和输出端之间的能量流动关系，其控制对象比单输入单输出变换器更复杂。不同端口的电压、电流和功率状态相互影响，控制策略不仅要维持输出稳定，还要兼顾光伏输入利用、电池充放电安全和功率器件热应力。与此同时，功率板中存在高 dv/dt 开关节点和较大的脉冲电流，对隔离驱动、采样抗干扰、PCB 布局和硬件保护提出了较高要求。因此，必须将主功率拓扑、采样链路、驱动链路和故障关断链路作为整体进行设计，而不能只关注单一控制算法。
\end{enumerate}

\subsection{创新点分析}

针对上述难点，本作品主要从系统架构、控制分工、协议组织和故障处理四个层面进行了设计，形成如下工程创新。

\begin{enumerate}[label=\arabic*、, leftmargin=*, itemsep=0.8ex, parsep=0pt, topsep=0.5ex]
  \item \textbf{多 MCU 分层协同架构。}  
  本作品采用 GD32G553、GD32C113 和 GD32VW553 三颗 MCU 协同工作的架构，将系统划分为主控层、电池管理层和无线交互层。主控侧负责统一调度、数据转发、PWM 输出和系统组织；BMS 侧负责电池测量、保护、均衡和 SOC 管理；无线侧负责远程监控与参数交互。该架构能够将高实时性任务与非实时任务分离，降低单芯片压力，提高系统稳定性。

  \item \textbf{主控侧四层解耦与发布/订阅总线机制。}  
  GD32G553 主控工程采用 APP / Middleware / Protocol / HAL 四层结构，将应用逻辑、数据总线、协议解析与底层驱动分离。其中心思想是通过统一调度器组织模块运行，通过发布/订阅总线解耦数据生产者和消费者，再通过协议层完成 CAN FD 解析和 UART 封装。该设计使得模块职责清晰、可迁移性较强，也便于后期增加新的采样项或转发项。

  \item \textbf{BMS 分层状态机与降级保护机制。}  
  GD32C113 BMS 工程并不是简单地“读数和输出”，而是采用具有明确状态边界的产品态机。系统在初始化阶段完成 AFE 唤醒、FET 关闭、均衡清零和基础配置；运行阶段依次执行测量、保护判断、SOC 更新、均衡控制和 FET 策略；一旦出现测量失败、保护触发或 FET 应用异常，则立即进入 DEGRADED 状态，执行均衡抑制和 FET 全关断。该机制体现了电池安全优先的设计原则，能够有效降低异常工况下的风险。

  \item \textbf{光储协同多模态能量调度策略。}  
  提出以光伏优先利用、母线稳定和电池安全为约束的能量调度逻辑，根据光伏功率、负载功率、电池 SOC 和故障状态自动选择工作模式，实现储能自适应充放电和离网稳压运行。

  \item \textbf{面向工程调试的可观测性设计。}  
  系统通过状态字、告警码、故障码、滚动计数器和运行日志等信息，提供了足够细粒度的可观测性。主控负责组织和转发，BMS 负责产生真实状态，无线侧负责呈现给用户。该设计使调试人员能够快速判断当前系统处于何种阶段、发生了何种异常以及异常来自哪一层，从而显著提高联调效率和演示稳定性。

  \item \textbf{控制与监控解耦的工程实现。}  
  传统方案常将控制、显示、记录和通信写在同一个主循环中，容易造成耦合过深、维护困难。本作品将控制链路和监控链路分开：控制链路以主控和 BMS 为核心，强调实时性和安全性；监控链路以无线模块为核心，强调可视化和交互性。该设计保证了即便监控端出现延迟或掉线，也不会影响底层保护和运行控制。
  
   \item \textbf{面向光伏、储能和输出端协同的三端口功率变换设计。}  
  本作品围绕光伏输入、电池储能和输出负载之间的能量流动关系，完成了三端口功率变换电路的仿真验证和板级硬件设计。与传统将光伏变换器、电池充放电变换器和输出变换器分立实现的方案相比，三端口结构能够在同一功率级内组织多端口能量传递关系，有利于减少级联变换环节，降低系统体积和后续扩展复杂度。该部分目前已完成主功率原理设计、采样调理设计、隔离驱动设计和 PCB 板级实现，后续将进一步开展实物闭环测试和额定工况验证。

  从主功率结构上看，功率板以光伏端、储能电池端和输出端为主要能量接口。原理图中设置了 PV、BAT 和 VOUT 三类功率端口，并围绕四绕组耦合电感构建能量传递路径。主功率开关器件包括 IRFP4668PBF、IPB073N15N5、BSC0702LS 以及 GC4D05120A 等器件，不同开关管承担不同端口之间的能量调制任务。通过多路 PWM 信号 \texttt{PWM\_M1}、\texttt{PWM\_MA}、\texttt{PWM\_M2} 和 \texttt{PWM\_M3} 的配合，系统可为后续实现光伏侧能量输入、电池侧充放电调节以及输出侧电压控制提供硬件基础。

  在采样与控制反馈方面，功率板针对光伏端和输出端设计了电压、电流和温度采样接口。电压采样包括 \texttt{V\_PV} 和 \texttt{V\_OUT}，电流采样包括 \texttt{I\_PV} 和 \texttt{I\_OUT}，温度采样包括 \texttt{TEMP\_MOS} 和 \texttt{TEMP\_MAG}。其中，电压采样前端采用高阻分压和运放缓冲结构，电流采样通过分流电阻与 INA181 电流检测芯片完成信号调理，温度采样用于监测功率 MOSFET 和磁性器件的热状态。这些采样量能够为后续 MPPT、充放电限流、输出稳压和热保护提供反馈依据。

  在驱动设计方面，功率板采用 UCC21540ADWKR 隔离驱动芯片完成主功率开关的栅极驱动，并为不同开关支路配置独立隔离电源。隔离驱动方式能够提高控制侧与功率侧之间的电气隔离能力，降低高 dv/dt 开关节点对 MCU 控制信号的干扰。驱动网络中设置了栅极电阻、下拉电阻、去耦电容和关断控制信号，用于改善开关速度、抑制误导通并增强故障状态下的关断可靠性。功率板还预留了 \texttt{OC\_FAULT} 等故障接口，可与主控制板的硬件保护输入配合，实现过流或异常状态下的快速关断。

  该三端口功率板的创新点不在于单个器件选型，而在于将主功率拓扑、采样反馈、隔离驱动、故障保护和 GD32G553 主控接口进行一体化设计。控制板负责产生 PWM 和处理采样反馈，功率板负责能量变换和功率器件驱动，BMS 负责电池侧安全约束，三者之间形成较完整的“控制-驱动-采样-保护”闭环基础。虽然当前功率板仍需进一步完成实物上电、闭环调试和长时间稳定性测试，但其仿真和板级设计已经为后续实现光储协同能量调度提供了硬件支撑。
\end{enumerate}
\subsection{本章小结}

本章围绕作品定位、实现难点和系统创新进行了说明。与单一控制器或单一功能模块方案相比，本作品的核心价值不在于堆叠功能，而在于通过多 MCU 分层协同、数据接口统一和分层保护机制，构建了一套结构清晰、职责明确、便于调试和展示的数字电源管理系统。后续章节将进一步展开系统总体方案、硬件设计、软件流程和测试验证等内容，以说明本作品的具体实现方式与工程效果。
