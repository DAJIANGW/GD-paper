## 一、论文审查

### 1. 整体印象

论文结构完整（摘要 → 目录 → 7 章正文 → 参考文献），格式规范，多 MCU 协同架构（G553 主控 + C113 BMS + VW553 无线）的分工表述清晰。BMS 部分（均衡计算、MOSFET 选型、分流电阻计算）写得相对扎实。

### 2. 核心问题 ⚠️

| 问题 | 说明 |
|------|------|
| **摘要与正文进度矛盾** | 摘要写"实现了光伏 MPPT 跟踪、光储多模态能量调度、离网稳压控制"，但正文多次声明"三端口功率板尚未完成完整实物闭环实现"。这是最大的问题，评委看到会质疑真实性。 |
| **三端口功率板内容过度重复** | 同样的内容（四绕组耦合电感、器件选型、采样调理、隔离驱动）在 1.3 创新点、2.5 节、2.6 节、3.4 节重复了 4 遍，需合并。 |
| **软件章节（第 4 章）严重薄弱** | 仅约 70 行，全是概述。缺少：主控四层架构模块图、BMS 状态机图、调度器流程、通信协议帧格式、MPPT 具体实现。且有未处理的 `\todo{}`。 |
| **测试章节缺核心数据** | 没有一张示波器波形、没有效率数据、没有动态响应曲线。所有测试结果表格全是"正常"二字，缺乏说服力。 |
| **图片大量缺失/重复** | 主功率电路原理图、软件流程图是 `\fbox` 占位框；多个 `figure` 共用同一 `label: fig:system_architecture`；缺 App 截图、缺实物照片。 |
| **参考文献不够支撑** | 缺少三端口/多端口变换器的核心 IEEE 文献来支撑你的功率拓扑方案选择。 |

### 3. 次要问题

- 第 2 章和第 3 章的"主控制板硬件设计"标题重复出现
- 英文摘要表达偏直译，部分句式生硬
- 一些公式推导（如 RC 滤波截止频率、均衡电流）属于教科书级内容，可以适当精简
- 第 2 章中三张主控板原理图使用了同一个 `\label{fig:system_architecture}`，需要分别改为不同的 label

## 二、修改思路

### 方向：调整论文定位

既然功率板未闭环，**不要把论文写成"完整光储系统已实现"**。建议重新定位为：

> **"基于国产 GD32 MCU 的多节点协同控制与电池管理平台——面向光储一体化电源的软硬件基础验证"**

这样论文重点变成：**控制平台搭建 + BMS 系统验证 + 无线监控联调 + 功率级硬件设计基础**，这是你已经完成且可以拿出证据的部分。

### 各章修改策略

| 章节 | 修改策略 |
|------|----------|
| **摘要** | 将"实现了 MPPT/能量调度/离网稳压"改为"设计了…控制平台，完成了…联调验证，搭建了…功率级硬件基础"。诚实但不降低工作量。 |
| **第 1 章（创新点）** | 保留多 MCU 协同、四层解耦、BMS 状态机三个核心创新。将三端口功率板创新点大幅压缩，改为"硬件平台预留与设计"一笔带过。 |
| **第 2 章（方案设计）** | 删除重复的三端口功率板描述，合并到一节。增加"为什么选三端口而非级联方案"的简要论证（需要文献支撑）。 |
| **第 3 章（硬件）** | 补充实际原理图截图、PCB 照片、实物照片。删除与第 2 章重复的段落。 |
| **第 4 章（软件）** | **重点扩充**。增加：① 主控四层架构模块调用关系图；② BMS 产品状态机状态转移图；③ CAN/UART 通信协议帧格式表；④ 调度器周期与任务优先级表。 |
| **第 5 章（测试）** | **重点扩充**。补充：① CAN 波形（示波器截图）；② BMS 数据串口日志；③ App/上位机实际截图；④ PWM 输出波形；⑤ 充放电过程的电压/电流/SOC 变化曲线。 |
| **第 6 章（结论）** | 调整措辞，强调"控制平台已完成验证，功率级硬件已设计完成待闭环测试"。 |
